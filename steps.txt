Step 1 — Start ngrok (Terminal 1)


ngrok http 8000
Copy the https://....ngrok-free.app URL it gives you.

Step 2 — Update .env if ngrok URL changed

Open .env and set:


NGROK_URL=https://your-new-ngrok-url.ngrok-free.app
(Skip if it's the same URL as before — free ngrok changes URL on every restart)

Step 3 — Activate venv + start server (Terminal 2)


cd C:\Users\Lenovo\Interview-bot
.venv\Scripts\activate
python app_full.py
You should see:


=======================================================
  PER-SESSION engine | concurrent interviews | dynamic questions
  ...
=======================================================
🗓️  scheduler worker started (poll every 30s)
🗄️  Supabase client ready
Step 4 — Deploy a bot (Terminal 2 or browser)

For a test with a pre-generated session:


GET http://localhost:8000/trigger-bot?meeting_url=YOUR_MEETING_URL&session_id=YOUR_SESSION_ID
For a quick test without session (uses static questions):


GET http://localhost:8000/trigger-bot?meeting_url=YOUR_MEETING_URL
That's it — 2 terminals total. ngrok stays running in T1, FastAPI in T2.

If you need to regenerate questions for a new candidate first, run this before step 3:


python test_eval.py --save
That prints a session_id you pass to /trigger-bot.
